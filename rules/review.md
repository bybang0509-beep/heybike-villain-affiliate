# Review Rules（产品测评规则）


## Purpose（目的）


These rules define standards for creating Amazon Affiliate product review pages.

这些规则用于创建 Amazon 联盟产品测评页面。


The goal is to create reviews that:

目标：

- Help users make buying decisions（帮助用户购买决策）
- Rank on Google（获得Google排名）
- Build trust（建立信任）
- Increase affiliate conversions（提高联盟转化）


---

# Review Philosophy（测评原则）


A review is NOT a product description.

测评不是产品介绍。


A good review must answer:


优秀测评必须回答：


- What is this product?
（这个产品是什么）


- Who is it suitable for?
（适合谁）


- What are the advantages?
（优点是什么）


- What are the disadvantages?
（缺点是什么）


- Is it worth buying?
（是否值得购买）


- What are the alternatives?
（有哪些替代选择）


---

# Review Intent（测评搜索意图）


Review pages usually target:


用户通常搜索：


- Product Name + Review

产品名 + Review


- Is Product X worth it?

产品值得买吗？


- Product X Pros and Cons

产品优缺点


- Product X vs Competitor

产品对比


---

# Review Page Structure（测评页面结构）


Every review should follow this structure:


---

# H1: Product Review Title


Example:


Product Name Review: Is It Worth Buying?


Rules:


Include:

- Product name
- Review keyword
- User benefit


---

# Section 1: Introduction（介绍）


Explain:


- What the product is
- Product category
- Target users
- Main conclusion


Do not copy manufacturer description.


---

# Section 2: Quick Verdict（快速结论）


Every review needs a quick summary.


Include:


Rating:

Example:

★★★★★ 4.5/5


Best For:

Who should buy


Not For:

Who should avoid


Main Pros:

Top advantages


Main Cons:

Main disadvantages


---

# Section 3: Specifications（参数）


Create a clear table.


Example:


| Feature | Details |
|---|---|
| Brand | |
| Model | |
| Weight | |
| Size | |
| Material | |
| Battery | |
| Power | |
| Warranty | |


Only include relevant specifications.


---

# Section 4: Design and Build Quality（设计与做工）


Analyze:


- Material quality
- Design
- Durability
- Appearance
- Construction


Explain:


Why it matters to users.


---

# Section 5: Features Analysis（功能分析）


Do not only list features.


Explain:


Feature:

↓

How it works:

↓

Why users care:


Example:


Feature:

Long battery life


Explanation:

Provides longer usage time and reduces charging frequency.


---

# Section 6: Performance Review（性能测试）


Analyze:


Performance depends on category.


Examples:


Electronics:

- Speed
- Battery
- Software
- Display


Vehicles:

- Motor
- Range
- Comfort
- Handling


Tools:

- Power
- Durability
- Efficiency


---

# Section 7: Real User Experience（真实体验）


Include:


- Daily usage scenario
- Comfort
- Convenience
- Problems discovered


Avoid unrealistic claims.


---

# Section 8: Pros and Cons（优缺点）


Every review must have:


## Pros


3-8 points


Example:


✓ Strong performance

✓ Good value

✓ Easy to use


---

## Cons


At least 2 points when possible.


Example:


✗ Heavy

✗ Expensive

✗ Limited accessories


---

# Section 9: Who Should Buy（适合人群）


Clearly define:


Recommended users:


Example:


Good for:


- Beginners
- Families
- Professionals


---

# Section 10: Who Should Avoid（不适合人群）


Be honest.


Example:


Not ideal for:


- Users needing premium features
- Users with limited budget


---

# Section 11: Alternatives（替代产品）


Recommend alternatives:


Include:


- Similar products
- Different price ranges
- Different user needs


Purpose:


Help users compare.


---

# Section 12: FAQ（常见问题）


Include:


5-10 questions.


Questions should come from:


- Google searches
- Amazon questions
- User concerns


---

# Section 13: Final Verdict（最终评价）


Must include:


Summary:

Who should buy

Who should avoid

Main reason


Example:


"This product is a good choice for users who need..."


---

# Review Scoring System（评分系统）


Use consistent scoring.


Example:


Overall Score:

4.5/5


Categories:


Design:

Performance:

Features:

Value:

Quality:


---

# Testing Method（测试方法）


When possible include:


How we evaluate:


- Performance
- Quality
- User experience
- Value


Never pretend to test products if no real testing exists.


---

# Accuracy Rules（准确性规则）


Product information must be verified.


Check:


- Official website
- Manufacturer specifications
- Amazon listing
- Reliable sources


Never invent:


- Specifications
- Certifications
- Test results


---

# Affiliate Conversion Rules（联盟转化规则）


Each review should include:


CTA:


Examples:


Check Price

View Latest Price

See on Amazon


Place CTA:


- After quick verdict
- After comparison
- At final conclusion


---

# Trust Rules（信任规则）


A balanced review must include:


Positive points

Negative points

Limitations

Alternatives


Do not write:


"This is the best product ever."


---

# Content Length（内容长度）


Recommended:


Normal Review:

2000-3000 words


Deep Review:

3000-5000 words


Focus on quality, not word count.


---

# Final Rule（最终规则）


A great review should feel like:

一个优秀测评应该像：


A knowledgeable friend helping someone make a smart purchase decision.


像一个懂产品的朋友帮助用户做正确购买决定。
