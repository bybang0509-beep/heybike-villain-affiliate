# QA Rules（质量检测规则）


## Purpose（目的）


These rules define quality assurance standards for Amazon Affiliate Websites.

这些规则用于规范 Amazon 联盟网站上线前的质量检查。


The purpose is NOT only to find technical errors.

目的不仅是发现技术错误。


The purpose is to ensure:

目标是确保：


- Users get real value（用户获得真实价值）
- Search engines can understand the website（搜索引擎可以理解网站）
- The website builds trust（网站建立信任）
- The website can generate sustainable traffic（网站可以长期获得流量）


---

# QA Philosophy（质量理念）


Before publishing any page, ask:


发布任何页面之前必须问：


"Does this page genuinely help a buyer make a better decision?"


“这个页面是否真正帮助购买者做出更好的决定？”


If the answer is no:

如果答案是否：

Improve the content before publishing.


---

# Website Quality Checklist（网站整体检查）


## Structure Check（结构检查）


Verify:


- Clear navigation
- Logical categories
- Correct page hierarchy
- No orphan pages


检查：


- 导航是否清晰
- 分类是否合理
- 页面层级是否正确
- 是否存在孤立页面


---

# SEO Quality Check（SEO检查）


Every important page must have:


## Metadata


Check:


- SEO Title exists
- Meta Description exists
- Correct keyword usage
- Open Graph configured


---

## Heading Structure


Check:


Correct:


H1

↓

H2

↓

H3


Rules:


- Only one H1
- No skipped hierarchy


---

## URL Check


Verify:


- Short URLs
- Clean structure
- Keyword relevance


Avoid:


Long random URLs


---

## Internal Linking Check


Every important page should have:


- Related articles
- Category links
- Product links


Check:


No broken internal links.


---

# Content Quality Check（内容质量检查）


## Originality


Verify:


Content is original.


Do not publish:


- Amazon copied descriptions
- Manufacturer copied text
- AI raw output


---

## Helpful Content Check


Ask:


Does this answer:


- What is this product?
- Who needs it?
- What are the benefits?
- What are the drawbacks?
- Is it worth buying?


---

## Buyer Pain Point Check


Check whether content addresses:


- Price concerns
- Quality concerns
- Performance concerns
- Reliability concerns
- User suitability


---

# Review Page QA（测评页面检查）


A review must include:


✓ Overview


✓ Quick Verdict


✓ Specifications


✓ Features


✓ Performance


✓ Pros


✓ Cons


✓ Who Should Buy


✓ Who Should Avoid


✓ Alternatives


✓ FAQ


✓ Final Verdict


---

# Comparison Page QA（对比页面检查）


Must include:


✓ Clear winner or recommendation


✓ Comparison table


✓ Important differences


✓ Different user recommendations


✓ Advantages and disadvantages


Do not:


Choose winner without explanation.


---

# Buying Guide QA（购买指南检查）


Must include:


✓ Buyer problems


✓ Selection criteria


✓ Common mistakes


✓ Product recommendations


✓ Alternatives


✓ FAQ


The guide should teach users how to choose.


---

# Affiliate Compliance Check（联盟合规检查）


Verify:


## Affiliate Disclosure


Must exist.


Must be clear.


---

## Recommendation Balance


Check:


Does the article include:


- Advantages
- Disadvantages
- Limitations


---

## Advertising Level


Avoid:


Too many buttons.


Too many affiliate links.


Aggressive sales language.


---

# Trust Check（信任检查）


Website should include:


- About page
- Contact page
- Affiliate disclosure
- Privacy policy
- Terms page


---

- Mobile performance

Mobile

Chrome

# Image QA（图片检查）


Every image:


Must have:


- Correct filename
- Alt text
- Optimized size


Check:


No broken images.


---

# Schema QA（结构化数据检查）


Verify:


Correct Schema:


- Product
- Review
- FAQ
- Article
- Breadcrumb


Check:


No validation errors.


---

# Accessibility QA（可访问性检查）


Verify:


- Alt text exists
- Buttons have labels
- Good contrast
- Keyboard navigation


---

# Security QA（安全检查）


Check:


No:


- Exposed API keys
- Sensitive data
- Unsafe scripts


---

# User Experience QA（用户体验检查）


Test:


Can a visitor quickly answer:


1. What is this product?


2. Is it suitable for me?


3. What are the advantages?


4. What are the disadvantages?


5. Should I buy it?


If not:

Improve layout or content.


---

# Final Publishing Checklist（上线检查）


Before publishing:


SEO:

□ Metadata completed

□ Keywords optimized

□ Schema added


Content:

□ Original

□ Helpful

□ Buyer focused


Design:

□ Mobile friendly

□ Professional design

□ Clear CTA


Technical:

□ Fast loading

□ No errors

□ Images optimized


Trust:

□ Disclosure added

□ Contact information

□ Legal pages


---

# Final Principle（最终原则）


A website should never be published because it is finished.


A website should be published because it is valuable.


不要因为页面完成就上线。


要因为它真正对用户有价值才上线。
