# Design System Tokens（设计系统）

Define and use consistent tokens:
定义并使用一致的设计标记：

- Colors: primary, secondary, accent, neutral (each with 50-900 scale)
  颜色：主色、辅色、强调色、中性色（各含 50-900 色阶）
- Typography: heading scale (h1-h6), body, caption
  字体：标题层级 (h1-h6)、正文、注释
- Spacing: 4px base unit (4, 8, 12, 16, 24, 32, 48, 64)
  间距：4px 基准 (4, 8, 12, 16, 24, 32, 48, 64)
- Border radius: sm, md, lg, full
  圆角：sm, md, lg, full
- Shadows: sm, md, lg
  阴影：sm, md, lg

## 尼尔森十大可用性原则

设计时必须遵守：

1. 系统状态可见性（加载状态、进度提示）
2. 系统与现实世界匹配（使用用户熟悉的语言和概念）
3. 用户控制和自由（撤销、返回、取消）
4. 一致性和标准（统一的交互模式）
5. 错误预防（防呆设计优于报错）
6. 识别而非回忆（减少记忆负担）
7. 灵活性和使用效率（快捷操作、自定义）
8. 美学和极简设计（不包含不必要的信息）
9. 帮助用户识别、诊断和修复错误（友好的错误提示）
10. 帮助和文档（必要的使用指引）

## 格式塔设计原则

- 接近性：相关元素靠近放置
- 相似性：相似元素视觉一致
- 连续性：视觉流引导用户注意力
- 封闭性：用户自动补全不完整图形
- 图形与背景：区分前景内容和背景

## 颜色对比度标准

### WCAG 2.1 AA
- 普通文本：4.5:1
- 大文本（18px+ 或 14px 加粗）：3:1
- UI 组件和图形对象：3:1

### WCAG 2.1 AAA
- 普通文本：7:1
- 大文本：4.5:1

## 动画性能优化

- 使用 transform 和 opacity 做动画（避免触发 layout）
- 避免布局抖动（layout thrashing）
- 使用 requestAnimationFrame
- 启用硬件加速（will-change / transform: translateZ(0)）
